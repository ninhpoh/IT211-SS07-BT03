
# 1. INPUT / OUTPUT

### Input
| Thành phần | Kiểu dữ liệu | Mô tả |
|---|---|---|
| `amount` | `double` | Số tiền giao dịch (rút tiền / chuyển khoản) |
| `toUser` | `String` | Tài khoản người nhận (chỉ dùng cho transfer) |
| `otp` | `String` | Mã xác thực OTP do người dùng nhập |

### Output
| Trường hợp | Kiểu trả về | Ví dụ |
|---|---|---|
| Thành công | `String` | `"Rút tiền thành công: -500000 VND"` |
| OTP null / rỗng | `OtpValidationException` | `"OTP không được để trống"` |
| OTP sai | `OtpValidationException` | `"Xác thực OTP thất bại"` |
| Không cần OTP | `String` | `"Số dư hiện tại: 2,000,000 VND"` |

---



##  LUỒNG XỬ LÝ

```
User gọi withdraw(amount, otp)
    │
    ▼
[Spring AOP Interceptor]
    │
    ├── Hàm có @RequiresOTP? ──No──▶ Thực thi bình thường
    │         │Yes
    │         ▼
    │   OTP null hoặc rỗng? ──Yes──▶ throw OtpValidationException("OTP không được để trống")
    │         │No
    │         ▼
    │   verifyOtp(otp) == false? ──Yes──▶ throw OtpValidationException("Xác thực OTP thất bại")
    │         │true
    │         ▼
    └──▶ proceed() → Thực thi logic kinh doanh → Trả kết quả
```
